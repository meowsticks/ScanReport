# AK ScanReport — `full-report-medium.html` Review Checklist

## Executive summary

This is a genuinely strong, near-polished **single-level v2 mockup**: the editor's focus-workspace, the shared `CORES` array driving pins + cards, the marking convention, the mandatory disclaimer, and the page-1 letterhead are all well-executed and internally consistent. But the document is **single-level by construction**, and the verdict on the central question is unambiguous: **the PDF layout does NOT work for P1 / L2 / L3.** Two independent walls block it — (a) there is no `level` field anywhere in the data model (`CORES`, targets, rebar, map all hard-code "P1"), and (b) every `.sheet` is a fixed `height:11in; overflow:hidden` box with hand-typed cards, so any content beyond the single P1 dataset is **silently clipped, not paginated.** Today's 9 cores happen to fit; tripling to ~27 cores plus L2/L3 maps would vanish off the bottom of the PDF with no warning. The first five things to fix, in order: **(1)** switch print pages from fixed-clipped to flowing (`height:auto; min-height; overflow:visible; page-break-after`), **(2)** add a `level` field to `CORES` + a `LEVELS` list and generate cards/pins/index per level, **(3)** add the global `print-color-adjust:exact` rule so the safety-critical colour code doesn't print grey, **(4)** reserve footer space / fix the absolute footer overlap, **(5)** fix the drag-select-hijack so report/core text is copyable. Everything else is polish.

## 1 — MULTI-LEVEL support (the report cannot represent P1/L2/L3 today)

- [ ] **`CORES` has no `level` field** · *high* — Level survives only as a text prefix in `loc` (`'P1 Bay 1 mid-span'`), so nothing downstream can group/subtotal/validate by floor. *Loc:* `CORES`, lines 1758–1786. *Fix:* add a required `level` to each core, drop the `'P1 '` prefix (derive display as `level + ' ' + loc`), and define a `LEVELS` array (`[{id:'P1',name,map,coverage},…]`) that map/pagination/subtotals key off.
- [ ] **Pages are fixed-height + hand-placed → 3 levels of cards get clipped** · *high* — Cards are literally typed into page 3 (A–E, 1247–1285) and page 4 (F–J, 1335–1364) inside `height:11in; overflow:hidden` sheets. The print side cannot grow. *Fix:* generate the card stream from `CORES.forEach` (the editor already does this at 1831–1838) into one flowing container; rely on `.card { break-inside:avoid }` (line 183) once pages flow (see group 4).
- [ ] **Only one slab plan exists; L2/L3 pins collapse onto the P1 map** · *high* — Single hard-coded map titled "…· P1 Slab" (line 1150) with all 9 pins (1172–1182); pins carry no level. *Fix:* loop `LEVELS` to emit one map block per level, filtering `CORES` by level; title/caption become `· {level} Slab` / `Figure N — {level} slab plan`. Normalised 0–100% x/y already make per-level reuse clean.
- [ ] **Cards/index/targets/rebar are not grouped or divided by level** · *med* (merge of 4 findings) — Card headers bury level in free text (1248…), the core index is one flat 9-row block (1200–1211), the targets table encodes level only inside the Location cell with no Level column (1300–1320), and the rebar mat summary is a single top/bottom card with no level label (1214–1231). *Fix:* group each by level with a `.level-divider` heading (`Scan Locations · L2 Deck`); add a **Level** column (or per-level `<tbody>`) to the targets table; repeat the `.matgrid` per level; namespace IDs (`P1-A`, `P1-T01`) so letters/T-numbers don't collide across floors.
- [ ] **Exec counts + footer page-totals are hand-typed and single-level** · *med* (merge) — Narrative "Nine…" (line 1114) and tiles 6/2/1/9 (1118–1123) are an all-levels roll-up with no per-level breakdown; every footer is a literal `Page X of 6` (1140, 1235, 1289, 1369, 1431, 1473). *Fix:* compute counts from `CORES` (`.length` + per-level `reduce`) so prose and tiles can't desync; add a per-level Safe/Caution/No-go/Total mini-table; replace `of 6` with CSS `counter(pages)` or a post-layout JS total, and add a `{level}` token to the running footer.
- [ ] **Editor has no level dimension** · *med* — Flat sidebar (1497–1511), single `#sec-map`/`#ed-pins` (1556–1575), one flat `#ed-cards` (1579), and hard-coded "9 cores" (1577) / "All 9 cores have verdicts" (1699); "Add core" (1491) can't say which level. *Fix:* add a level switcher (tabs or nav group) that filters map + card list; add a Level selector to each core card and to Add-core; make the readouts compute per-level + project-wide off the now-level-tagged `CORES`.
- [ ] **Per-level area/coverage & level-qualified uncertainty notes** · *low* (merge) — "~64 m² across 3 levels" (1102) and "100% of marked areas" (1108) are lumped; uncertainty notes cite "Bay 4 NE corner" / "Bay 3 SW edge" (1383, 1386) with no floor, ambiguous when every deck has a Bay 3/4. *Fix:* itemise area/coverage per level (keep the rolled-up total) and prefix each uncertainty item with its level (`L2 · Bay 4 NE corner`).

## 2 — Copy/paste & interaction bugs

- [ ] **Drag-selecting card/report text is hijacked into focusing the card, then the text is hidden** · *high* — The global handler (line 1876) only listens for `click`; a press-drag-release inside a summary still fires `click` → `focusCard()` runs `scrollIntoView({behavior:'smooth'})` (1873) and `.find.focused .summary .obs{display:none}` (914) removes the text mid-selection. Same root cause leaves the printed `.sheet` copy-safe only by luck. *Fix:* at the top of the 1876 handler bail on an active selection (`var s=getSelection(); if(s&&s.type==='Range'&&!s.isCollapsed) return;`) and/or ignore the focus branch if the pointer moved >4px since `mousedown`; add `user-select:text` to `.summary, .summary *` and to `.sheet, .sheet *`.
- [ ] **Two document Escape handlers double-fire** · *med* — Line 1926 always `blurAll()`; line 2030 closes the AI panel only when open. With the slide-over open, Escape closes it **and** collapses the card being edited underneath. *Fix:* in the 1926 handler `if(aiPanel && aiPanel.classList.contains('open')) return;`, or consolidate to one handler with explicit precedence.
- [ ] **AI backdrop closes the panel mid-copy; bubbles aren't explicitly selectable** · *med* — `.ai-msg .b` (1037) has no `user-select`; a drag that starts in a bubble and ends over `#ai-backdrop` (pointer-events:auto when open, 1022) hits its `closeAssistant` (2013). *Fix:* add `user-select:text; cursor:text;` to `.ai-msg .b`; in the backdrop handler ignore the click when a selection exists and/or only close when `e.target===aiBack`.
- [ ] **First click on a collapsed card's field smooth-scrolls the card before the caret settles** · *med (verify)* — On a not-yet-focused card the handler calls `focusCard()` and returns (1901–1904); `scrollIntoView({behavior:'smooth'})` (1873) plus the collapsed→expanded `.fields` `max-height` transition (915–916) move the target field while the user is aiming, so caret placement feels unreliable. (Native focus *does* land on first click — the original "two clicks required" framing is too strong.) *Fix:* drop `behavior:'smooth'` or use `block:'nearest'`; if the click landed on `input,textarea,button,select`, don't fight native focus.

## 3 — Scan-card alignment (editor + print)

- [ ] **Verdict pill text sits ~2.5px high in the collapsed card head** · *med* — `.editor .find .head { align-items:center }` (line 568) centres a 13px title box against a 9.5px pill, so their text baselines diverge; the printed `.card .hd` already uses `align-items:baseline` (188), so the two views disagree. *Fix:* change line 568 to `align-items:baseline`.
- [ ] **Wrapped recommendation has no hanging indent (print)** · *med* — In `.card .vd` the pill and recommendation share one inline flow; when the rec wraps (Core E no-go, 1283) line 2 slides back under the pill. *Fix:* on `.card .vd` (line 198) add `padding-left:58px; text-indent:-58px;`, or float the `.pill` and drop the `&nbsp;`.
- [ ] **"Improve with AI" button floats mid-row, mixed-case, in the editor** · *med* — `.ai-open` is inline after the label word with only `margin-left:8px` (1015–1018), so it sits far left of the field's right edge and reads as "OBSERVATION ✨ Improve with AI". *Loc:* template 1820–1821. *Fix:* make `.find .f > label { display:flex; align-items:baseline; justify-content:space-between; }` and drop the `margin-left`.
- [ ] **Editor summary concatenates obs+rec; print splits them into two rows** · *low* (merge of 2) — Editor renders `obs … <b class="rec">rec</b>` as one `.obs` line (1807) and `.rec` has no CSS, so the single most actionable line gets no lift; print uses separate `.obs` + `.vd` rows. *Fix:* mirror print — emit a separate `.vd` row with the verdict tag + rec, and add `.find .obs .rec { display:block; margin-top:5px; color:var(--ed-acc-2); font-weight:700; }`.
- [ ] **Long card titles will push the size text off baseline (print)** · *low (verify)* — `.card .hd` (185) has no `min-width:0`/wrap control on the title node; all 9 current titles fit one line, so latent only. *Fix:* wrap the title in a `<span class="t">` with `min-width:0`, set `.card .hd { align-items:flex-start }`, and `.card .hd .r { white-space:nowrap; flex:none; padding-top:1px; }`.

## 4 — Print / page mechanics

- [ ] **Fixed `height:11in; overflow:hidden` silently CLIPS overflow** · *high* (the single biggest print risk; root of several others) — Base `.sheet` (70–78) and the print rule (1056–1060) re-assert `height:11in` without resetting `overflow`, so it inherits `hidden`. *Fix:* in `@media print` set `.sheet { height:auto; min-height:11in; overflow:visible; page-break-after:always; padding:0.55in 0.6in; }` and let Chrome paginate, leaning on the existing `break-inside`/page-break rules.
- [ ] **Absolutely-positioned footer overlaps body content on full pages** · *high* (merge of 3) — `.ft { position:absolute; bottom:0.32in }` (253–260) and its `::before` micro-line at `bottom:100%` (693–697) are out of flow; nothing reserves the ~0.55in band, so the last card/`.sign`/sign-off prints over them. Pages 2 and 5 are most exposed, worse once multi-level data is added. *Fix:* bump `.sheet { padding-bottom:0.95in }`; long-term, once pages flow, move the footer + micro-line to `@page { @bottom-center{…} }` running elements.
- [ ] **Colour code can print grey — `print-color-adjust:exact` is set only on the gold header name** · *high* — Line 117 is the only place; the verdict pills (208–210), map pins (290–302), marking swatches/dots (752–764), the red no-scan hatch (757–758, 769) and tendon/conduit lines (779–780) all rely on `background-color`, which Chrome drops unless "Background graphics" is on — collapsing the safety-critical convention to white. *Fix:* add `@media print { * { -webkit-print-color-adjust:exact !important; print-color-adjust:exact !important; } }`.
- [ ] **`@page { margin:0 }` makes correct output depend on Chrome dialog defaults** · *med* — Line 1052 pushes to the paper edge; at default margins + "Headers and footers" on, Chrome shifts every page and can push the footer off-page. The on-screen hint (1076) doesn't enforce it. *Fix:* set `@page { size:letter; margin:0.5in }` and reduce `.sheet` padding to match, so output is right even at defaults.
- [ ] **`.matgrid` (and `.sign`) lack `break-inside:avoid`** · *med* (merge) — `.card/.mapwrap/.disclaimer/.review` carry it (183/270/683/792) but `.matgrid` (334–337) + its PT note (1229) and `.sign` (372–376) do not, so they can split mid-block once pages flow. *Fix:* add `break-inside:avoid` to `.matgrid` and `.sign`; wrap the Rebar H2 + grid + note in one keep-together div.
- [ ] **Page 2 is over-stuffed — closest to the clip edge even single-level** · *med* — Map (320px ≈ 3.33in, line 277) + caption + legend + marking key + 9-row index + rebar H2 + matgrid + note + footer nearly fill the ~9.9in usable area (1145–1237). *Fix:* trim the print map to ~260–280px and/or split Rebar Mat onto its own page; resolves naturally once pages flow.
- [ ] **Continuation pages 2–6 carry no project identity except the 7.5pt footer** · *low* — By design the letterhead is page-1 only (138–140, 1148). *Fix:* add a slim 8pt continuation band (project no. + title + page) to pages 2–6, or a running `@page` top element once flowing.

## Quick wins (do these first — highest value, lowest effort)

1. **Global print colour rule** — add `@media print { * { -webkit-print-color-adjust:exact !important; print-color-adjust:exact !important; } }`. One line; saves the entire safety colour code from printing grey.
2. **Footer clearance** — bump `.sheet { padding-bottom:0.95in }` (line 74) so the absolute footer + micro-line stop overlapping body content.
3. **Card-head baseline** — change `.editor .find .head` `align-items:center` → `baseline` (line 568); matches print, removes the 2.5px pill drift.
4. **Split the two reds** — repoint `--alert` (line 19) off `#c0282d` so the disclaimer/alert red is no longer identical to the conduit marking red.
5. **One shared `.ed-card .actions` rule** — fixes the Targets left-alignment, the 4/10/12px Done-button jump, and lets you delete six inline styles in a single edit (and add `.find .head align-items:baseline` companion + label-weight unify while you're there).

*Note:* the two true blockers for multi-level (add a `level` field to `CORES`; switch print pages to flowing/`overflow:visible`) are high-value but not low-effort — schedule them as the first real implementation tasks immediately after the quick wins.
